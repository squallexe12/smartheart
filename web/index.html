SmartHeart estimates the probability of coronary artery disease from thirteen routine clinical signals. The model is a logistic regression hand-derived in NumPy — no scikit-learn shortcuts — then frozen, serialized, and shipped to your browser. Every weight is auditable. Every prediction is local.

    02 How it works

    Linear, then squashed.

    Logistic regression in one sentence: multiply each input by a learned weight, sum, then squash the total through a sigmoid to get a number between zero and one.

    01

    Encode

    Six numeric vitals are standardized using the training mean and standard deviation. Categorical fields are one-hot encoded with the first category dropped as a reference level.

    02

    Score

    A dot product against eighteen frozen weights produces a single real number, the logit. Each summand is a transparent contribution attributable to one signal.

    03

    Squash

    The sigmoid maps the logit into a probability. Bigger logits push the output toward one; very negative logits push it toward zero.

    f(x) Hypothesis
    p = σ ( w · x + b ) = 1 1 + e−(w · x + b)

    The eighteen-dimensional vector x is the encoded patient. The vector w and scalar b were learned via batch gradient descent minimising binary cross-entropy on 238 training samples.

    03 About the model

    UCI Heart Disease, the Cleveland subset.

    Trained on 238 patients from the cleanest of the four UCI hospital cohorts. Tested on a held-out 59. The full implementation — gradient derivation, training loop, evaluation — lives on GitHub.
